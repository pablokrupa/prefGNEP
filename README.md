# prefGNEP
### A Python library for learning generalized Nash equilibria using a preference-based active learning method

This repository contains a package for learning an approximate **Generalized Nash Equilibrium** (GNE) of a **Generalized Nash Equilibrium Problem** (GNEP) using pairwise preference information.
The method is appropriate for settings where players' best responses and cost functions are unknown and cannot be evaluated directly — only binary pairwise preference observations are available (*"I prefer strategy A over strategy B"*).

Pairwise preferences are collected using an Active Learning (AL) loop, where cost functions of a surrogate GNEP are fitted to the observed preferences, and new query points are generated by an AL exploitation-exploration strategy.
The approach requires no gradient information from the true cost functions and is applicable to settings with general shared constraints (box, linear inequality/equality).

**prefGNEP** relies on the [NashOpt](https://github.com/bemporad/nashopt) package for solving the underlying (surrogate) GNEP at each iteration of the AL loop, and uses [JAX](https://github.com/jax-ml/jax) for efficient gradient-based optimization.

## Overview

**prefGNEP** considers a GNEP with $N$ agents, where each agent $i$ solves the optimization problem

$$
\begin{aligned}
x_i^\star(x_{-i}) \in \arg\min_{\ell_{i}\leq x_{i}\leq u_{i}} &f_i(x)\\
\textrm{s.t.} \quad &g(x) \leq 0 \\
&A_{\textrm eq}x = b_{\textrm eq}\\
&h(x) = 0
\end{aligned}
$$

where:
- $f_i \colon \mathbb{R}^{n} \to \mathbb{R}$ is the objective of agent $i$, specified as a [JAX](https://github.com/jax-ml/jax) function;
- $x = (x_1^\top \dots x_N^\top)^\top \in \mathbb{R}^n$ are the decision variables, $x_i\in\mathbb{R}^{n_i}$, $\sum_{i=1}^N n_i = n$. We write $x = (x_i, x_{-i})$ to denote the decision variables of agent $i$ and the other agents;
- $g : \mathbb{R}^n \to \mathbb{R}^{n_g}$ encodes shared inequality constraints (JAX function);
- $A_{\textrm eq}, b_{\textrm eq}$ define linear shared equality constraints;
- $h : \mathbb{R}^n \to \mathbb{R}^{n_h}$ encodes shared nonlinear equality constraints (JAX function);
- $\ell_{i}, u_{i}$ are local box constraints.

A GNE $x^*$ is a point in which no individual agent has an incentive to change its decision, given the decisions from other agents, i.e.,

$$f_i(x^{\*}_{i}, x^{\*}_{-i}) \leq f_i(x_i, x^{\*}_{-i})$$ 

for all feasible $x=(x_i,x_{-i}^*)$.

**prefGNEP** learns an approximate GNE $x^*$ by iteratively fitting surrogate cost functions $\hat{f}_i$ of a GNEP using preferences that are selected using an AL strategy that balances exploitation (querying near a GNE of current surrogate GNEP) and exploration (querying in unknown regions).

## Basic Usage

The core workflow involves three steps:

1. **Define** the surrogate model structure (e.g., via `gen_quad_models`) and instantiate `PrefGNEP`;
2. **Generate** an initial dataset of pairwise preference observations;
3. **Run** the active learning loop (`fit_AL_loop`) to iteratively refine the surrogates and converge towards a GNE of the underlying GNEP.

A squematic code outline is as follows:

```python
from src.pref_gnep import PrefGNEP
from src.models import gen_quad_models

# 1. Define problem
sizes = [n1, n2, n3]  # Number of decision variable sizes of each player
lb = ...  # Lower bounds for decision variables
ub = ...  # Upper bounds for decision variables
Aeq = ... # Linear equality constraint matrix
beq = ... # Linear equality constraint vector
def g(x, p=0.0): ...  # Shared inequality constraints (JAX function)
def h(x, p=0.0): ...  # Shared nonlinear equality constraints (JAX function)

# 2. Generate surrogate functions, with signature f(th_i, x_i, x_mi, p=0.0).
# The package provides a utility function gen_quad_models() for generating quadratic
# surrogate models, but users can also define their own surrogate functions.
# I nitial parameter guess `th` and lower and upper bounds `th_min` and `th_max` are required.
th, fs, th_min, th_max = gen_quad_models(sizes, full=False, diagonal=False)

# 3. Define true cost functions for the oracle, such that f_real_i(xA) <= f_real_i(xB) iff the
# oracle prefers strategy xA over xB for player i. The function should have signature f_real_i(x).
def f_real_1(x): ...  # JAX functions
def f_real_2(x): ...
def f_real_N(x): ...
f_real = [f_real_1, f_real_2, ..., f_real_N]

# 4. Set up PrefGNEP
pg = PrefGNEP(sizes, fc=fs, lb=lb, ub=ub, g=g, ng=ng, h=h, nh=nh, Aeq=Aeq, beq=beq)
pg.init(th=th, th_min=th_min, th_max=th_max)  # Initialize surrogate parameters and bounds
pg.loss(rho_th=1e-3)  # Initialize hyperparameters of loss function
pg.optimization(adam_epochs=500, adam_eta=0.001)  # Initialize optimization hyperparameters

# 5. Generate initial dataset
ds = pg.generate_initial_dataset(f_real, n_samples=50)

# 6. Run the AL loop. Optionally provide hyperparameters (see method's docstring for details).
ds, info = pg.fit_AL_loop(f_real, ds, n_iters=100, sigma=0.3, delta=0.5)

# 7. Retrieve the estimated GNE
x_star = pg.solve_gnep().x
```

Some notes:
- Surrogate functions `fc` have signature `f(th_i, x_i, x_mi, p=0.0)`, where `th_i` are the parameters of the surrogate cost function for player `i`, `x_i` are the decision variables of player `i`, and `x_mi` are the decision variables of the other players. The parameter `p=0.0` is included for compatibility with [NashOpt](https://github.com/bemporad/nashopt).
- Functions `g` and `h` also require a parameter `p=0.0` for compatibility with [NashOpt](https://github.com/bemporad/nashopt).
- In `fit_AL_loop()`, argument `n_iters` is the numer of iterations of the AL loop, `delta` is the exploration hyperparameter, and `sigma` is the noise hyperparameter. Additional hyperparameters are explained in the method's docstring.

For complete, self-contained examples covering a range of problems from the literature, refer to the scripts in the [`examples/`](examples/) directory.
